## 2. 背景与痛点 (0:30 - 2:00)

**[幻灯片内容]**
*   **当前测试流程的挑战**:
    1.  **需求文档(PRD)非结构化**: 包含大量文字、流程图、UI截图，人工阅读理解慢。
    2.  **用例编写效率低**: 从理解需求到编写用例，耗费测试人员30%-50%的时间。
    3.  **一致性难以维护**: 需求变更后，用例更新滞后。
    4.  **评审成本高**: 依赖人工评审，缺乏客观的质量评估标准。

**[演讲稿]**
在传统的软件开发流程中，测试人员面临着巨大的挑战。首先，PRD文档通常是图文混排的，非结构化信息多，理解成本高。其次，人工编写用例效率低，且容易遗漏边界条件。最后，如何客观评价用例质量、如何确保用例与需求的一致性，一直是一个难题。我们的项目正是为了解决这些问题而生。

---

## 3. 解决方案概览 (2:00 - 3:30)

**[幻灯片内容]**
*   **核心理念**: AI驱动的自动化测试生成管线
*   **流程图**:
    `飞书PRD文档` -> `[多模态解析]` -> `[结构化数据]` -> `[LLM推理生成]` -> `[测试用例]` -> `[智能对比评估]`
*   **关键能力**:
    *   ✅ **飞书深度集成**: 一键导入在线文档。
    *   ✅ **多模态理解**: 识别流程图、UI设计稿（Doubao Vision）。
    *   ✅ **思维链生成**: 深度分析业务逻辑（Doubao Thinking）。
    *   ✅ **闭环评估**: 自动对比人工用例与AI用例。

**[演讲稿]**
AI TestGen Pro 提供了一站式的解决方案。我们通过飞书开放平台API直接获取PRD文档，利用多模态大模型（Doubao Vision）识别文档中的图片内容（如流程图和UI稿），将其转化为文字描述。然后，利用具备强推理能力的模型（Doubao Thinking）深度分析业务逻辑，自动生成覆盖前端交互、后端接口及异常场景的测试用例。最后，我们还引入了智能评估机制，对生成结果进行打分。

## 4. 核心技术亮点 (3:30 - 6:00)

**[幻灯片内容]**

### 4.1 多模态文档解析
*   **技术**: Feishu Open API + Doubao Vision (seed-1.6-vision)
*   **创新点**:
    *   突破纯文本限制，精准识别图片中的业务流程和界面元素。
    *   **自动降级策略**: 优先使用视觉大模型，失败时自动降级为OCR识别，确保高可用性。
    *   **并发处理**: 多线程并行处理文档图片，大幅提升导入速度。

### 4.2 智能用例生成
*   **技术**: Prompt Engineering + Doubao Thinking (seed-1.6-thinking)
*   **策略**:
    *   **结构化提取**: 将PRD解析为前端模块、后端接口、业务规则等结构化数据。
    *   **场景覆盖**: 强制要求覆盖 20% 异常场景、15% 边界值场景。
    *   **标准输出**: 直接生成可执行的 JSON 格式用例。

### 4.3 智能对比评分系统
*   **技术**: 混合匹配算法 + LLM 加权评分
*   **逻辑**:
    1.  **初筛**: 基于关键词和Jaccard相似度快速匹配候选用例。
    2.  **精评**: 调用 LLM 从四个维度进行加权打分：
        *   语义一致性 (30%)
        *   步骤逻辑 (30%)
        *   预期结果 (20%)
        *   数据准确性 (20%)

**[演讲稿]**
我们的系统有三大技术亮点。
第一是**多模态解析**。我们不仅能读文字，还能"看懂"图片。我们集成了 Doubao Vision 模型，能够精准识别 PRD 中的流程图步骤和 UI 元素。为了保证稳定性，我们还设计了 OCR 降级方案。
第二是**智能生成**。我们不只是简单的文本续写，而是要求模型先将 PRD 结构化，再基于测试方法论生成用例，并强制要求覆盖异常和边界场景。
第三是**智能评分**。这是我们的一大特色。我们实现了一套"混合匹配+LLM评分"的机制，能够像资深测试专家一样，从语义、逻辑、数据等多个维度，对 AI 生成的用例进行客观打分，让质量看得见。
